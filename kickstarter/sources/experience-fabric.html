Design principles

Locked where it matters.

A few things are non-negotiable so the experience is predictable everywhere — the parts users build muscle memory around.

HomeAgentsProjectsDevicesVault
One visible agent: THOXY
Only THOXY is user-visible by default; every other agent routes through it unless Advanced Mode is on. One face, one voice, one place to ask.
Locked navigation
Exactly five sections, in order, on every certified surface — desktop sidebar or mobile bottom bar. Never added, removed, renamed, or reordered.
Four-tier memory
HOT (this conversation) · WARM (this project) · COLD (retrievable history) · VAULT (sealed, encrypted). Same model on the smallest device and the largest.
Voice state ladder
idle → listening → thinking → responding | executing → completed | error → idle. The same states, wherever THOXY listens.
Why it matters

Scientific by design.

TXF is a fully-adopted internal development framework — not a mood board. Consistency is measured and gated, so it holds under change.

Tokens are generated
never hand-edited
Design tokens compile from one authored source through the UX engine; a CI drift gate fails the build if any surface falls out of sync.
Four gates on every PR
brand · WCAG AA · contracts · drift
Brand rules, WCAG AA contrast, component contracts, and token drift are checked automatically before anything ships.
Plain-language specs
standards/
UX, interaction, accessibility, digital-human, and agent-behavior specifications — the contract the whole company builds to.
≥ 90
The THOX Experience Score. Every release is scored 0–100 by the certification release gates. 90+ ships, 80–89 gets review, below 80 is blocked. Certified bundles carry a txf.json manifest with an experience-score floor of 90.
