# Demo Acceptance Checklist

Use this checklist before approving any Kickstarter device demo clip.

## Universal approval gate

| Status | Requirement | Owner |
|---|---|---|
| [ ] | Product names are correct | Producer |
| [ ] | Prices match `docs/REWARDS_MATRIX.md` | Craig |
| [ ] | Founder titles are correct | Producer |
| [ ] | No real credentials or private data appear | Tommy |
| [ ] | Generated visuals are not presented as final production photography | Craig |
| [ ] | Local-first language is accurate | Tommy |
| [ ] | Delivery or manufacturing claims are approved | Craig |
| [ ] | Clip has captions | Producer |
| [ ] | Clip is readable on mobile | Producer |
| [ ] | Audio is clean | Producer |

## Rejection reasons

Reject or reshoot the clip if any of these appear:

- A price is wrong.
- A legacy product name appears.
- Real credentials or personal data appear.
- ThoxKey is framed as a standalone model-running computer.
- ThoxMini Air is framed as a high-end AI workstation.
- ThoxClip is framed as medical, emergency, safety, or surveillance hardware.
- A generated render is framed as final production photography.
- Text is unreadable on mobile.
- Speaker title is wrong.
- The demo creates a promise that the fulfillment plan cannot support.
